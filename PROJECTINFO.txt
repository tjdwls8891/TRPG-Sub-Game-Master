==============================================================
TRPG Sub Game Master — Claude 세션 온보딩 레퍼런스 (v1.7)
==============================================================
Gemini API + discord.py 기반 한국어 TRPG 보조 GM 디스코드 봇.
GM이 마스터 채널에서 명령어를 입력하면 AI가 묘사를 생성하고
비용 추적·캐시 관리·기억 압축·BGM/이미지 연출을 자동화한다.

==============================================================
1. 파일 맵
==============================================================
main.py         TRPGBot(commands.Bot 상속). active_sessions{ch_id: session},
                session_io_locks{sid: Lock}, playlist_sessions{guild_id: mgr},
                genai_client, system_instruction, intro_text.
                setup_hook()에서 cogs/*.py 자동 로드.
                on_ready()에서 restore_sessions_from_disk(bot) 호출.

core.py         전역 상수, 데이터 모델, 유틸리티 함수, UI 클래스 전담.

cogs/session.py  !새세션, !시작, !소개
cogs/game.py     !진행, !주사위, !재생성, !기억압축, !노트, !캐시노트
                 on_message(자동 로그), on_typing(GM 타이핑 동기화)
cogs/character.py !참가, !설정, !증감, !외형, !프로필, !엔피씨, !설정생성
cogs/media.py    !이미지, !브금, !플리, !볼륨, !채팅
cogs/system.py   !명령어, !채널정리, !세션종료, !캐시, !리로드

scenarios/       *.json — 읽기 전용 정적 시나리오 데이터
media/           media/{scenario_id}/*.mp3, *.png — 오프라인 에셋
sessions/        sessions/{8자리UUID}/ — 동적 세션 데이터 (런타임에 생성됨)
                  data.json, api_log.txt, game_chat_log.txt,
                  master_chat_log.txt, cost_log.txt
paused_session/  수동으로 이동해 둔 세션 백업본 (자동으로 복구되지 않음)

3. 핵심 알고리즘 / 패턴
==============================================================

[A] PromptBuilder.build_prompt() 주입 순서 (core.py)
  ① add_memory_block()      — compressed_memory (절대 참조용 누적 기억)
  ② add_note_block()        — session.note (GM 하드코딩 노트)
  ③ add_player_block()      — players 전체 스탯·외형·resources·statuses
  ④ add_triggered_npc_block() — 최근 10개 raw_logs + current_turn_logs + gm_instruction
                               에 이름이 등장 AND default_npcs와 details가 다를 때만 주입
  ⑤ add_keyword_memory_block() — keyword_memory의 keywords가 위 combined에 있을 때만 주입
  ⑥ add_recent_action_block() — current_turn_logs 전체
  ⑦ add_gm_instruction_block() — clean_instruction
  ⑧ add_rule_enforcement_block() — 최종 룰 강제 + status_code_block 출력 지시

[B] 캐시 패딩 해킹 (build_scenario_cache_text)
  Gemini 캐싱 최소 32,768토큰 요건 우회.
  base_tokens < 32768 → missing_tokens * 4개의 '.'을 "[System Data Padding Area - DO NOT READ]"
  헤더와 함께 룰북 끝에 삽입. cache_note는 룰북 [6] 섹션 하단에 지연 병합.

[C] !진행 태그 파싱 (game.py — proceed_turn)
  정규식으로 태그 추출 후 instruction에서 제거한 clean_instruction만 AI에 전달.
  - 상/중/하:키워드  → top_imgs / mid_imgs / bottom_imgs 큐에 적재
    상: 첫 문단 출력 후 전송 | 중: 텍스트에 키워드 등장 시 동적 삽입 | 하: 묘사 끝 후 전송
  - 자:이름;아이템;수치  → session.resources[이름][아이템] += 수치
  - 태:이름;상태  → session.statuses[이름].append(상태)
  - 태:이름;-상태 → session.statuses[이름].remove(상태)

[D] !시작 role 조작 해킹 (session.py)
  start_message를 types.Content(role="model") 로 raw_logs에 삽입.
  → AI가 "내가 이 묘사를 했다"고 인식 → 이후 GM 스탠스 자동 유지.
  is_started = True 로 중복 실행 차단.

[E] 기억 압축 (game.py — proceed_turn / compress_memory)
  turn_count % 5 == 0 시 자동 트리거. !기억압축으로 강제 실행 가능.
  uncompressed_logs → build_compression_prompt(session, log_text) → LOGIC_MODEL
  → 응답을 compressed_memory에 append, uncompressed_logs에서 del.

[F] !재생성 롤백 (game.py — regenerate_turn)
  ① game_channel.purge(after=anchor_msg, check=bot.user) — 이전 턴 봇 출력 삭제
  ② raw_logs[-2:]에서 user 콘텐츠 분석 → "\n[GM 지시]:" 기준 split → current_turn_logs 복원
  ③ raw_logs = raw_logs[:-2] / uncompressed_logs = uncompressed_logs[:-2]
  ④ turn_count -= 1 / last_turn_anchor_id = None
  turn_count % 5 == 0인 경우(압축 직후) 롤백 차단.

[G] 캐시 자가복구 (game.py — generate_with_retry)
  APIError(400/404, "cache" in str) 발생 → process_cache_deletion(보관비 정산)
  → build_scenario_cache_text → caches.create → retry_count=1로 재귀 호출.
  retry_count=1 상태에서 재실패 시 예외 그대로 raise.

[H] 스트리밍 연출 (core.py — stream_text_to_channel)
  text.split('\n\n') → 문단별 새 메시지 전송 → words_per_tick(기본 5)단어씩
  asyncio.sleep(tick_interval=1.5) 후 message.edit() → 타이핑 연출.
  문단이 1950자 초과 시 break. game_chat_log에 최종 텍스트 기록.

[I] 세션 직렬화/복구
  save_session_data: raw_logs → [{role, text}] 변환 후 data.json 덮어씌움.
                     asyncio.Lock으로 동시 쓰기 방지.
  restore_sessions_from_disk: data.json → TRPGSession 재조립 → cache_name ping 테스트
                               → 만료 시 자동 재발급 → active_sessions[game_ch_id/master_ch_id] = session.

4. 비용 모델 (core.py)
==============================================================
EXCHANGE_RATE = 1500.0  (KRW/USD 고정)

PRICING_1M 모델별 단가 (USD/100만 토큰):
  gemini-3-flash-preview:  INPUT=0.50, OUTPUT=3.00, CACHE_READ=0.05, STORAGE/hr=1.00
  gemini-3.1-pro-preview:  INPUT=2.00, OUTPUT=12.00, CACHE_READ=0.20, STORAGE/hr=4.50
  gemini-2.5-pro:          INPUT=1.25, OUTPUT=10.00, CACHE_READ=0.20, STORAGE/hr=4.50

IMAGE_GEN_COST = 0.04 * 1500 = ₩60 (1024x1024 1장 고정)

calculate_upload_cost(model, in, out, cached_read):
  actual_input = in - cached_read
  → (actual_input * INPUT + out * OUTPUT + cached_read * CACHE_READ) / 1M * EXCHANGE_RATE

calculate_storage_cost(model, tokens, duration_seconds):
  minutes = round(duration_seconds / 60)  ← 초→분 반올림
  → tokens / 1M * (STORAGE/hr / 60) * minutes * EXCHANGE_RATE
  max cap: 21600초(6시간)

process_cache_deletion(bot, session):
  보관비 정산 → session.cache_name/obj/model/created_at/tokens 초기화 → save

write_cost_log → sessions/{id}/cost_log.txt 에 타임스탬프+사용처+토큰+비용 기록

7. 현행 시나리오: shadow_island (영도 — 고립된 낙원)
==============================================================
현재 버전: v1.7 개발 진행 중 (v1.6 기반)

설정 개요:
  배경: 2016년 부산 영도구. 바이러스로 이성을 잃은 '살아있는 인간' 감염자.
  모든 교량 차단, 해로 차단. PC는 쪽배를 타고 영도에 막 상륙한 본토 생존자.

pc_template: {능력, 체력, 근력, 민첩, 기술, 소지품(기본값: "생수 1병, 과자 약간, 쇠파이프")}

판정 시스템: D20 롤 언더 (결과 ≤ 스탯+가중치 = 성공). 평균 능력치=10, PC범위 5~15.
  대성공(1,2) / 대실패(19,20) — 기준치 4이하 or 17이상 시 예외조항 적용.

4대 세력 및 핵심 NPC:
  남항 중공업 연대 (조선소, 북서쪽)
    강철수(58, 의장) 김영호(26, 천재기술자/희망호 제작중) 정미란(46, 병참국장)
    한태식(50대, 수습대장) 박두식(40대, 현장의사/해부연구)

  해련 자치 마을 (봉래산 해련사+빌라촌)
    지홍(60대, 주지/창고관리) 강현식(50대, 스카이웨이 설계자) 박주현(50대, 경작대/농업교수)
    이민수(20대, 청년수송대장)

  조도 방위 사령부 (한국해양대 조도캠퍼스)
    박진우(40대, 함장/원칙주의) 서하준(20대, 육상작전대장/내적갈등) 정은주(50대, 생명공학교수/치료파)
    최기철(50대, 군수담당관/밀거래)

  중리 교역소 (동삼1동, 중립 회색지대)
    송미숙(40대, 중재자/정보통) 잭(30대, 송 마담 직속 수거꾼/묵묵) 오창석(60대, 장부/신용평가)
    김수진(20대, 해안경비팀 리더/작살)

감염자 특성: 살아있는 인간. 잔존 행동(문고리 흔들기, 열쇠 찾기 등). 소리/빛에 반응.
              세계관상 치료 가능성 연구 중(정은주).

8. 개발 주의사항 / Gotchas
==============================================================
- DEFAULT_MODEL = LOGIC_MODEL = "gemini-3-flash-preview" (Pro는 주석 처리됨)
  비용 계산은 PRICING_1M 딕셔너리 기준이므로 모델 변경 시 pricing 키도 확인 필요.

- bot.active_sessions에 game_ch_id 와 master_ch_id 양쪽 모두 같은 session 객체를 등록함.
  채널정리 시 _cleanup_session_memory()가 두 키 모두 pop 해야 메모리 누수 없음.

- !이미지 생성은 scenario.json 파일을 직접 덮어씀 (media_keywords 영구 추가).
  이미지 생성 시 scenario_data 메모리와 파일 모두 동시 갱신.

- !재생성은 turn_count % 5 == 0 일 때 차단 (압축 직후 롤백 불가).
  압축이 이미 uncompressed_logs를 지웠으므로 롤백 기준이 없음.

- stream_text_to_channel은 "> " prefix 자동 추가. paragraph가 ">"로 시작하면 prefix 생략.
  game_chat_log에 최종 텍스트(final_text[:2000])만 기록됨.

- on_typing: is_processing=True 중에는 타이핑 동기화 스킵. 마스터 메시지 전송 시 gm_typing_task.cancel().

- 캐시 model 속성: session.cache_model = core.DEFAULT_MODEL 로 저장.
  process_cache_deletion에서 getattr(session, "cache_model", DEFAULT_MODEL)로 안전 접근.

- !세션종료와 !캐시 삭제 모두 genai_client.caches.delete 실패해도 process_cache_deletion으로
  내부 메타데이터는 반드시 초기화 (try-except Fallback 구조).

- paused_session/은 자동 복구 대상이 아님. sessions/ 로 옮겨야 restore_sessions_from_disk 적용.
